## MVP 代码质量保证方案 V2

- 因用户不会写代码，原方案"核心模块人工手写"不可行，重新设计为"全 AI 编码 + 可解释交付物"方案
- 核心思路转变：从"谁写"到"怎么验"，代码由 AI 写，但"对不对"用非代码方式证明
- 三层防线：①自动化（TS+ESLint+构建阻断）②AI 流程控制（可解释交付物+双人 AI 交叉验证）③产品经理验收（测试用例+流程图+真实场景）
- 可解释交付物铁律：核心模块必须同步输出 ①完整单元测试 ②文本流程图 ③边界条件清单，缺一不验收
- 核心模块稳定化机制：DRAFT → STABLE → FROZEN，STABLE 模块变更需先输出变更申请
- 红线判定：5 种情况触发立即回滚（缺交付物、测试不足、流程图矛盾、场景失败、未经申请改 STABLE）
- Roadmap 配套更新：§5.8 质量门禁、§5.8 知识传递风险、§7.7 风险矩阵「AI 生成代码质量不可控"

## 甘特图组件设计与外部库调研

- **现有实现扫描**：纯自研，约 1,177 行代码（136 TSX + 660 CSS + 102 详情面板 + 92 容器 + 187 数据层）
- **功能盘点**：已有 12/16 项功能 ✅（时间轴、任务条、里程碑、分组、选中态、关键路径、图例、统计、详情面板、响应式、sticky 滚动）；缺失 4 项 ❌（依赖线可视化、拖拽调整、时间粒度切换、今天标记线）
- **外部库调研**：对比 dhtmlx-gantt、gantt-task-react、frappe-gantt、vis-timeline、syncfusion-react-gantt
  - dhtmlx：功能最全，但暗色主题需重写 300+ CSS 规则，商业版 $599/年
  - gantt-task-react：原生 React、轻量，但不支持分组折叠和关键路径高亮
  - syncfusion：功能最全 + 内置暗色主题，但免费版有水印，$995/年
  - frappe-gantt：极简，不满足需求
- **核心结论**：不建议引入外部库。现有实现已完美匹配暗色玻璃态设计规范，外部库适配成本高于收益
- **建议路线**：保持自研，分阶段补充 4 个缺失功能
  - P0（1 天）：今天标记线 + Hover Tooltip
  - P1（2 天）：依赖线可视化（SVG 连线层）+ 时间粒度切换（日/周/月/季）
