# 用户旅程分析

## 概述

用户旅程分析帮助你理解用户在使用产品过程中的完整体验路径。不只是功能列表，而是用户从"第一次接触"到"熟练使用"的情绪、行为和痛点变化。

## 何时使用

- 设计新功能前，需要理解用户当前的使用路径
- 发现用户流失点，需要定位问题环节
- 规划产品 onboarding 流程
- 评估功能优先级时，需要判断对用户体验的影响
- 产品经理要求你做用户视角的分析

**何时不使用：** 纯粹的技术实现讨论、后端 API 设计、数据库 schema 设计。

## 核心方法

### 1. 用户画像 (Persona)

明确你的目标用户是谁：

- 角色、目标、痛点
- 技术水平、使用场景
- 对产品的期望

### 2. 触点映射 (Touchpoint Mapping)

列出用户与产品的所有交互点：

- 首次接触 → 注册 → 首次使用 → 核心功能 → 深度使用
- 每个触点的用户情绪（😊 满意 / 😐 中性 / 😞 沮丧）

### 3. 阶段模型 (Phase Model)

将用户旅程划分为阶段：

- 认知期：用户如何发现产品
- 上手期：用户如何完成首次核心任务
- 习惯期：用户如何将产品融入日常工作
- 传播期：用户如何推荐给他人

### 4. 痛点与机会

每个阶段识别：

- 当前痛点
- 改进机会
- 优先级建议

## 输出格式

完成后输出一份简洁的用户旅程报告：

1. **用户画像** — 1-2 段描述目标用户
2. **旅程地图** — 阶段 → 触点 → 情绪 → 痛点
3. **改进建议** — 按优先级排列，每条建议对应一个具体痛点
