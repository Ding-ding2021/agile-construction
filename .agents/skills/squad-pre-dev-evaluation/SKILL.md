# Squad Pre-Dev Evaluation

## 概述

接到任何开发任务时，**先评估任务风险等级**，再根据等级决定是否触发评估组。

| 等级评估结果  | 处理方式                              |
| ------------- | ------------------------------------- |
| **L1 低风险** | 跳过评估组，组长单人代检后直接开发    |
| **L2 中风险** | 触发精简评估（组长 + 1 名评估员）     |
| **L3 高风险** | 触发全量评估（组长 + 3 名评估员并行） |

**硬性门禁**：L2/L3 未全票通过，不得进入开发阶段。

## 触发条件

- **任何新需求、新功能、新项目 — 先评估等级，再决定是否走评估流程**
- 重大重构或架构变更之前
- 用户需求模糊时（由 brainstorming skill 衔接到此）
- L1 不触发评估组，由组长单人完成评估后直接进入开发

## 流程

### Step 1: 确定任务风险等级

| 等级 | 判断标准                 | 评估组配置           |
| ---- | ------------------------ | -------------------- |
| L1   | 单文件/纯样式/文档       | 免评估，组长单人代检 |
| L2   | 新增组件/单页面/小功能   | 组长 + 1 名评估员    |
| L3   | 跨多文件/架构/新数据模型 | 全量 4 人            |

### Step 2: 并行派发子代理

根据风险等级派发：

```
L3 全量：
  ├── agent-1: 产品评估员（v4-flash）
  ├── agent-2: UI/UX 评估员（v4-flash）
  ├── agent-3: 技术评估员（v4-pro，含测试策略）
  └── agent-0: 组长（v4-pro）—— 同步等待，后续汇总

L2 精简：
  ├── agent-1: 评估员（v4-flash，任选一个视角）
  └── agent-0: 组长（v4-pro）

L1 单人：
  └── agent-0: 组长（v4-flash/go）
```

### Step 3: 各子代理输出独立报告

每个子代理必须输出自己的独立评估报告，不可共享状态。

**产品评估员输出模板：**

- 需求合理性：需求是否清晰、有价值
- 用户场景：真实用户如何使用
- 建议：如有改进建议

**UI/UX 评估员输出模板：**

- 设计规范一致性：是否符合 design-spec-v2-shadcn.md
- 交互模式：是否符合项目惯例
- 可访问性：WCAG 2.1 AA 基本要求
- 建议：如有改进建议

**技术评估员输出模板：**

- 可行性：技术上是否可行
- 影响范围：哪些代码/模块会受影响
- 技术债务：是否引入或暴露了技术债务
- **测试策略概览（必填）：**
  - 单元测试：核心逻辑/工具函数，预计用例数
  - 组件测试：关键交互场景，预计用例数
  - **E2E 测试：前端功能必须有 Playwright E2E 覆盖，覆盖核心用户流程**
  - 边界条件：空/极限/异常输入场景列表
  - 验收标准：build/lint/test/e2e 全部通过
- 建议：如有改进建议

### Step 4: 组长汇总

1. 收集全部独立报告
2. 逐条判断是否通过
3. 输出汇总评估报告（使用 agent-squad-protocol.md 中的模板）
4. 如有 CHANGES REQUESTED，合并到设计文档中
5. 如有 BLOCKED，说明原因并升级给用户

### Step 5: 输出设计文档

组长产出设计文档，包含：

- 架构设计
- 数据流
- 组件结构
- 测试策略（来自技术评估员）
- 路由设计

### Step 6: 全票通过后移交

评估通过后，通知用户确认，然后：

- 调用 `writing-plans` skill 拆分实现计划
- 将计划和评估报告移交给开发交付者

## 铁律

- **评估未全票通过，不开工**
- **子代理各自输出独立报告，不共享状态**
- **技术评估员必须包含测试策略概览**
- **不跳过 L2/L3 任务的评估流程**

## 常见错误

- L2/L3 任务因"感觉简单"跳过评估
- 产品评估和 UI 评估合并为一个人，丢失视角差异
- 评估通过后不做设计文档就直接开工
- 测试策略写成"待定"而非具体用例数

## 输出物清单

- [ ] 产品评估报告
- [ ] UI/UX 评估报告
- [ ] 技术评估报告（含测试策略）
- [ ] 组长汇总评估报告
- [ ] 设计文档（含测试策略）
- [ ] 实现计划（由 writing-plans 产出）
