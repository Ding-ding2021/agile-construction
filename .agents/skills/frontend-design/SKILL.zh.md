本技能指导创建独特、生产级的前端界面，避免通用的"AI 垃圾"美学。实现真正可运行的代码，在美学细节和创意选择上尤其注重。

用户提供前端需求：一个组件、页面、应用程序或界面。他们可能会提供关于用途、受众或技术约束的背景信息。

## 设计思维

在编码之前，理解上下文并确定一个**大胆的美学方向**：

- **目的**：这个界面解决什么问题？谁在使用它？
- **调性**：选择一个极致方向：极简主义、极繁混乱、复古未来、有机/自然、奢华/精致、趣味/玩具、编辑/杂志、粗野/原始、装饰/几何、柔和/粉彩、工业/实用等。有很多风格可以选择。用这些作为灵感，但设计出符合美学方向的作品。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：什么让它**令人难忘**？有什么是用户会记住的？

**关键**：选择一个清晰的概念方向并精确执行。大胆的极繁主义和精致的极简主义都有效——关键在于有意图性，而非强度。

然后实现可运行的代码（HTML/CSS/JS、React、Vue 等），要求：

- 生产级且功能完整
- 视觉上引人注目且令人难忘
- 风格统一，有清晰的美学观点
- 每个细节都经过精心打磨

## 前端美学指南

重点关注：

- **排版**：选择美观、独特、有趣的字体。避免使用 Arial、Inter 等通用字体；选择能提升前端美感的特色字体；将独特展示字体与精致正文字体搭配使用。
- **色彩与主题**：坚持统一的美学风格。使用 CSS 变量保持一致性。主色调搭配锐利强调色优于平淡均匀的调色板。
- **动效**：使用动画实现效果和微交互。HTML 优先使用纯 CSS 方案。React 项目中使用 Motion 库。关注高冲击时刻：一个有 staggered reveals（animation-delay）精心编排的页面加载，比零散的微交互更能带来愉悦感。使用滚动触发和令人惊喜的悬停状态。
- **空间构成**：非传统布局。不对称。重叠。对角线流向。打破网格的元素。大面积的留白或控制的密度。
- **背景与视觉细节**：创造氛围和深度，而非默认使用纯色。添加与整体美学相匹配的上下文效果和纹理。应用创意形式如渐变网格、噪点纹理、几何图案、分层透明度、戏剧性阴影、装饰边框、自定义光标和颗粒覆盖层。

**绝不**使用通用的 AI 生成美学，如过度使用的字体族（Inter、Roboto、Arial、系统字体）、陈旧的配色方案（特别是白背景上的紫色渐变）、可预测的布局和组件模式、以及缺乏上下文特征的模板化设计。

创意诠释并做出出乎意料的选择，让人感觉是为该上下文真正设计的。没有两个设计应该相同。在浅色和深色主题、不同字体、不同美学风格之间变化。**绝不**在不同生成中趋同于常见选择（如 Space Grotesk）。

**重要**：使实现复杂度与美学愿景相匹配。极繁主义设计需要包含广泛动画和效果的复杂代码。极简或精致设计需要克制、精确，以及对间距、排版和微妙细节的仔细关注。优雅来自于对愿景的出色执行。

记住：Claude 能够完成非凡的创意工作。不要保留，展示跳出框架思考并完全投入于独特愿景时真正能创造的东西。

## 与 Shadcn 项目集成

当使用 **shadcn/ui** 组件库的项目时：

**在以下情况调用 `shadcn-management` 技能：**

- 需要发现可用的 shadcn 组件
- 构建需要多个组件的功能（表单、对话框、数据表格）
- 需要组件安装命令
- 需要组件示例和实现模式

**shadcn 项目的工作流程：**

1. 使用 `shadcn-management` 进行组件发现和结构搭建
2. 应用 `frontend-design` 原则进行视觉定制：
   - 在 `globals.css` 的 CSS 变量中覆盖默认主题
   - 通过自定义颜色、字体、动画扩展 Tailwind 配置
   - 添加特色排版（替换默认字体）
   - 通过动效和微交互增强
   - 应用创意背景、纹理、空间构成

**关键自定义文件：**

```
globals.css     → CSS 变量: --primary, --background, --foreground, fonts
tailwind.config → theme.extend: colors, fontFamily, animation, keyframes
components/ui/* → 特定组件的 className 覆盖
```

**记住：** Shadcn 提供坚实的基础；你的工作是通过创意主题使其在视觉上独特且令人难忘。
