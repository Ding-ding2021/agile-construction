# 文档合约同步

## 概述

本技能强制执行**三层文档架构**：每次创建或修改人类文档时，必须同步生成对应的 AI 合约文件。合约文件是面向 AI Agent 的结构化摘要——纯表格、清单、键值对，不包含叙事段落。

**核心理念**：人类看完整叙事，AI 看结构化合约。两个读者、两份文档、双向映射。

---

## 三层架构

```
人类文档层（docs/）      AI 合约层（docs/ai/）      记忆与进化层（memory/）
─────────────────────    ─────────────────────    ─────────────────────
读者：人类               读者：AI Agent            读者：AI（跨会话）
格式：段落 + 叙事         格式：表格 + 清单          格式：键值对 + 日志
长度：不限               长度：≤ 200 行 / 文件      长度：持续追加
写入者：各角色            写入者：产品（本技能）      写入者：产品（task-memory）

数据流向：
  人类写 PRD → 本技能提取合约 → AI 读合约执行任务
                                    ↘ 完成任务 → 提取模式 → 反哺 knowledge/
```

---

## 触发条件

本技能在以下任何场景触发：

- 创建或修改 `docs/01-product/` 下的 PRD
- 创建或修改 `docs/02-architecture/` 下的架构文档
- 创建或修改 `docs/00-governance/` 下的治理文档（涉及规则变更时）
- 用户要求"审核文档""提取合约""同步文档""生成 AI 合约"
- 其他技能（如 `documentation-and-adrs`、`spec-driven-development`）完成后级联调用
- `docs/README.md` 索引更新后

**不触发**：纯 `04-operations/` 运营报告、`99-archive/` 归档操作、每日日志写入。

## 工作流程

```
收到文档变更信号
  │
  ▼
┌────────────────────────────────────────────┐

└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 2：判定是否需要合约                     │

│ - 03-engineering/ 指南   → 视内容判断         │
│ - 04-operations/ 报告    → 不生成             │
└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 3：从人类文档提取结构化内容              │
│ 必须提取以下部分（如文档中缺失则留空）：        │
│ - 模块定位（一句话）                         │
│ - 核心实体及字段                             │
│ - 状态机规则                                │
│ - 业务规则（编号列表）                        │
│ - 依赖模块（模块名 + 引用段落）               │
│ - API 骨架（方法和路径）                      │
│ - 质量门禁                                  │
└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 4：写入 AI 合约                        │
│ - 路径：docs/ai/contracts/<模块名>.md         │

│ - 使用下方模板，严格遵守格式                  │
│ - 写入后立即检查行数，超 200 行则拆分或精简     │
└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 5：建立双向链接                         │
│ - 人类文档 frontmatter 增加：                 │
│   ai_contract: docs/ai/contracts/<name>.md  │
│ - 合约 frontmatter 填写：                    │
│   human_source: docs/01-product/<name>.md   │
└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 6：更新索引                             │
│ - docs/ai/README.md 登记新合约               │
│ - 如有新模式 → docs/ai/knowledge/patterns.md │
│ - 如有新决策 → memory/MEMORY.md              │
└────────────────────────────────────────────┘
  │
  ▼
┌────────────────────────────────────────────┐
│ 步骤 7：合约自检（硬性门禁，不过不准收工）      │
│ - [ ] 行数 ≤ 200？                          │
│ - [ ] 无叙事段落？                           │
│ - [ ] 每节都是表格/清单/键值对？               │
│ - [ ] frontmatter 完整？                    │
│ - [ ] human_source 指向有效路径？             │
│ - [ ] 已登记到 docs/ai/README.md？           │
└────────────────────────────────────────────┘
```

## AI 合约模板

**所有合约必须使用此模板，不得自由发挥格式：**

```markdown
---
id: AI-<MODULE-NAME>
human_source: docs/01-product/<source-file>.md
status: active
last_synced: YYYY-MM-DD
---

# AI 合约：<模块中文名>

## 模块定位

<一句话描述，不超过 50 字>

## 核心实体

| 实体     | 字段                 | 状态机         |
| -------- | -------------------- | -------------- |
| <实体名> | <逗号分隔的字段列表> | <状态流转描述> |

## 业务规则

1. <规则>
2. <规则>

## 依赖模块

| 模块     | 引用位置       | 依赖内容       |
| -------- | -------------- | -------------- |
| <模块名> | <人类文档段落> | <具体依赖什么> |

## API 骨架

| 方法 | 路径            | 说明   |
| ---- | --------------- | ------ |
| POST | /api/<resource> | <说明> |

## 质量门禁

- <门禁规则>
```

## 格式约束（硬性红线）

以下规则**不可突破**，违反任何一条 = 合约不合格：

| #   | 规则                                                             | 检测方式                                        |
| --- | ---------------------------------------------------------------- | ----------------------------------------------- |
| 1   | 合约文件 ≤ 200 行                                                | `wc -l docs/ai/contracts/<name>.md`             |
| 2   | 零叙事段落 — 不允许"在 x 模块中，我们设计了…"这类描述性文字      | 人工目视：文中不得出现连续 3 句以上自然语言段落 |
| 3   | 每个章节必须是表格、编号列表或键值对                             | 人工目视：检测到段落体立即修正                  |
| 4   | frontmatter 必须完整（id / human_source / status / last_synced） | 人工目视                                        |
| 5   | human_source 必须指向存在的文件                                  | `test -f docs/01-product/<name>.md`             |
| 6   | 合约文件必须登记到 `docs/ai/README.md`                           | `grep <name> docs/ai/README.md`                 |

## 与记忆层的协作

本技能完成合约同步后，如涉及以下情况，还需写入记忆层：

| 情况             | 动作                                           |
| ---------------- | ---------------------------------------------- |
| 发现新的代码模式 | 追加到 `docs/ai/knowledge/patterns.md`         |
| 架构决策变更     | 追加到 `docs/ai/knowledge/decisions.md`        |
| 新的不可违背规则 | 追加到 `docs/ai/knowledge/rules.md`            |
| 影响全局架构     | 更新 `memory/MEMORY.md`                        |
| 每日任务完成     | 调用 `task-memory` 写入 `memory/YYYY-MM-DD.md` |

---

## 错误恢复

| 异常                          | 处理                                       |
| ----------------------------- | ------------------------------------------ |
| 人类文档 frontmatter 不完整   | 先补齐人类文档 frontmatter，再继续         |
| 人类文档状态为 draft          | 跳过，不生成合约（草稿不应有合约）         |
| 合约超过 200 行               | 检查是否有叙事段落混入；或拆分为两个子合约 |
| human_source 指向的文件不存在 | 确认人类文档已保存，再重试                 |
