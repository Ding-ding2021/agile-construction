两种交付策略

方案 A：分阶段（推荐）

阶段 1 — WBS 树核心

  • 扩展 SubTaskTree → WBS 编码+进度汇总
  • 工作包 CRUD + 工期/前置任务字段
  • 数据导出 → 思维导图/甘图输入

阶段 2 — 甘特图 + 网络图

  • 集成 Gantt 库（@svar-ui/react-gantt）
  • 依赖连线 + 关键路径
  • 网络图基于 same data 自动排布

阶段 3 — 思维导图画布

  • 轻量画布 React Flow / 自绘
  • 拖拽创建节点 → 一键导入 WBS 树

方案 B：树+甘特同步开发

一阶段完成树 + 甘特

  • WBS 树核心功能（编码/汇总/CRUD）
  • 甘特图同时到位（使用第三方库）
  • WBS 树 ↔ 甘特图双向联动
  • 工期/依赖在树中编辑，甘特图实时反映
  • 网络图和思维导图延后
  • ⚠ 单次交付量大，约 2-3 个工作日

各视图技术选型

视图 技术方案 理由
🌳 树 自建（基于现有 SubTaskTree） 已用 shadcn/ui Table，避免引入重量级树库
📊 甘特图 @svar-ui/react-gantt 或自建 高星（134★），React19/TS，支持 10K 任务
🔗 网络图 @xyflow/react (React Flow) DAG 自动布局 + 自定义节点，社区成熟
🧠 思维导图 React Flow 或 markmap React Flow 复用网络图技术栈，或 mindmap 轻量方案
