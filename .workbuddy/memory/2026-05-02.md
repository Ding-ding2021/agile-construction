## 设计规范组件实例化（TaskDetailPage 试点）

### 关联 Issue

- 无（技术债务清理）

### 问题/需求

设计规范 `docs/00-governance/design-specification.md` 中定义的原子组件（card、button、input、select）停留在文档层面，未落地为代码。TaskDetailPage 直接裸用 MUI 原始组件，视觉样式与规范不一致。

### 修改内容

- **新建组件**:
  - `PmCard.tsx`：标准卡片容器（bg/border/radius/padding/shadow 对齐规范）
  - `PmCardHeader.tsx`：卡片标题（图标 + 标题，gap/margin 对齐规范）
  - `PmSelect.tsx`：规范样式下拉选择，下拉菜单自定义 Paper 样式（深色背景、MenuItem 悬停/选中态对齐品牌色）
- **增强组件**:
  - `PmButton.tsx`：primary 增加 `--pm-shadow-primary` 阴影；secondary 重写为规范样式（蓝边框透明底）；danger 使用品牌玫红
  - `PmInput.tsx`：覆盖 border-radius/bg/border/height/focus 态；type="date"/"datetime-local" 时的 height 固定 36px 和日历图标滤镜处理
- **修正颜色**:
  - `toneColors.ts`：fallback 回退值改为品牌色（`#16D924`、`#D9165E`、`#45A9E8`）
- **迁移试点（TaskDetailPage）**:
  - `Button` → `PmButton`、`TextField` → `PmInput`、`Select`/`FormControl` → `PmSelect`、局部 `FormField` → `PmInput readOnly`、部分 `Paper` → `PmCard`
  - 标签输入从文本输入框+按钮改为 MUI Autocomplete 多选选择器（Chip 展示，规范样式）
  - 计划开始/结束改用 PmInput type="date"
  - 实际开始/结束改用 PmInput type="datetime-local"（readOnly）
