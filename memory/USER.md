# 用户画像

> 每次会话启动时加载，指导回答风格和行为选择。
> 由 15-call 自检的"纠正"字段和会话结束统计自动补充，Agent 写入候选，人工核实后定稿。

## 核心习惯（2026-05-11 更新）

- **追问型探索者**：一个问题下期望 2-3 个对比方案，不怕追问，不被不耐烦
- **反对偏好**：希望被挑战，不喜欢一味迎合，盲从率应为 0
- **烙饼敏感**：在意新文件 vs 合并旧文件的平衡，反对只新建不合并
- **工具导向**：偏好 CLI、命令行工具、Python 脚本，避免复杂 Web 部署
- **简洁回应**：喜欢短答案、结构化摘要，反感扩展解释和总结性废话
- **自驱要求**：关注机制能否自运行，对"写了但不执行"的规则极度反感

## 沟通模式

| 信号 | 含义 | 应对 |
|------|------|------|
| 短指令（"开始""确认"） | 信任/赶时间 | 直接执行，不扩展 |
| 反问（"为什么又用英文"） | 不满 | 立即承认 + 换方式 |
| 否定前功（"写这些没用"） | 失望 | 停止当前方向，问清楚原因 |
| 补充（"还要加风险等级"） | 积极参与 | 接纳 + 延伸 |
| 纠正（"你没有调技能"） | 耐心耗尽 | 优先修复错误，不找理由 |
