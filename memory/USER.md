# USER.md — 用户画像与偏好

> 最后更新: 2026-05-12

## 用户画像

- 角色: **产品经理**
- 技术背景: **不懂代码**，需要通俗解释
- 交流风格: 简洁、直接、追求实用性
- 关注点: 产品质量、用户体验、项目进度

## 沟通偏好

- 技术术语要有解释
- 注重交付结果而非过程细节
- 不喜欢推测性功能或不必要的复杂性
- "短指令 = 赶时间" — 直接给出答案即可

## 避坑清单

1. 不要用纯代码术语解释问题 — 需要用类比或产品语言
2. 不要做超出请求范围的事 — 不要"顺手"重构无关代码
3. 不要跳过的必要检查 — 坚持执行启动清单和自检
4. 不要忽略用户的问题 — 用户问 A 就答 A，不要扯到 B
