# V1 全量交付开发规划 V3.0

> **规划性质**: V1 全量交付 — 补齐所有 P0/P1 模块 + 全部债务清偿 + 全场景 AI Agent  
> **基线版本**: 基于 Development Plan V2.1（Phase 1~5 已完成）  
> **交付标准**: V1 Product Definition（PRD-012）全部交付判定项  
> **执行策略**: 混合模式 — 先清偿债务加固底座 → 分批补齐模块 → 最后 Agent + 联调验证  
> **节奏**: 稳扎稳打 — 按依赖顺序，每阶段结束后质量门禁

## 2. 开发阶段划分

### 整体路线图

```
Phase A: 底座加固与债务清偿 ───────────────────────────────────
  A1 甘特图拖拽编辑
  A2 依赖类型扩展
  A3 撤销/重做
  A4 基线对比
  A5 RBAC 权限框架（框架级，模块级后续落地）
    │
Phase B: 模块功能补齐 ─────────────────────────────────────────
  B1 人员管理补齐（角色管理 + 用户-角色映射）
  B2 系统设置补齐（权限配置 + 用户管理 + 审计日志）
  B3 工队管理补齐（任务派单联动）
  B4 验收管理建设（验收计划 + 检查项 + 整改追踪）
  B5 采购管理补齐（审批流程 + 预算联动）
  B6 标准管理补齐（版本控制 + 验收派生闭环）
    │
Phase C: Agent 能力建设 ───────────────────────────────────────
  C1 Agent 节点框架（触发 + 执行 + 确认 + 留痕）
  C2 Agent 目录管理界面
  C3 品牌需求 Agent
  C4 项目经理 Agent
  C5 验收质检 Agent
  C6 AI 辅助标准条款提取
    │
Phase D: 工作台与联调验证 ────────────────────────────────────
  D1 工作台首页重构（待办 + 预警 + 动态 + 快捷入口）
  D2 项目级 RBAC 权限落地
  D3 端到端状态机联调
  D4 种子数据初始化与治理
  D5 V1 交付验收
```

> **目标**: 清偿全部 6 项技术债务，为后续模块补齐奠定交互和数据基础  
> **前置条件**: Phase 1~5 已完成  
> **质量门禁**: 债务项完成后回归测试原有功能不受影响

## 3. 任务依赖关系与状态机

### 3.1 阶段依赖总线

```
Phase A (底座加固)
  ├── A1 (甘特图拖拽) ─── 独立，可并行
  ├── A2 (依赖类型) ─── 独立，可并行
  ├── A3 (撤销重做) ─── 独立，可并行
  ├── A4 (基线对比) ─── 依赖 A1（甘特图交互）
  └── A5 (RBAC框架) ─── 独立，可并行（但后续所有模块依赖它）
      │
Phase B (模块补齐) ─── 依赖 A5（RBAC 框架）
  ├── B1 (人员管理) ─── 依赖 A5
  ├── B2 (系统设置) ─── 依赖 A5 + B1
  ├── B3 (工队管理) ─── 独立
  ├── B4 (验收管理) ─── 独立
  ├── B5 (采购管理) ─── 独立
  └── B6 (标准管理) ─── 依赖 B4（验收派生）
      │
Phase C (Agent) ─── 依赖 Phase B
  ├── C1 (Agent框架) ─── 独立启动（可提前与 B 并行）
  ├── C2 (目录管理) ─── 依赖 C1
  ├── C3 (品牌需求Agent) ─── 依赖 C1 + C2
  ├── C4 (项目经理Agent) ─── 依赖 C1 + C2
  ├── C5 (验收质检Agent) ─── 依赖 C1 + C2 + B4
  └── C6 (标准条款提取) ─── 依赖 C1 + B6
      │
Phase D (联调验证) ─── 依赖 Phase B + C
  ├── D1 (工作台) ─── 依赖 B1~B5（需要各模块数据）
  ├── D2 (RBAC落地) ─── 依赖 A5 + B1 + B2
  ├── D3 (联调) ─── 依赖全部
  ├── D4 (种子数据) ─── 依赖全部
  └── D5 (验收) ─── 依赖全部
```
