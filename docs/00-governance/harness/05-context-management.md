# Harness 上下文管理

## Clause 1. 上下文管理目标

**1.1 [强制]** 确保 AI Agent 在每次会话中拥有完成任务所需的全部上下文。

**1.2 [强制]** 避免上下文膨胀导致质量下降。

**1.3 [强制]** 每 15 条用户消息执行一次上下文自检。

---

## Clause 2. 上下文分层

**2.1 [参考]** 上下文按使用频率分三层：

| 层     | 内容                              | 加载时机 | 典型大小   |
| ------ | --------------------------------- | -------- | ---------- |
| 常驻层 | 角色 Persona、项目规则、AGENTS.md | 会话启动 | ~5 KB      |
| 任务层 | 当前任务相关的 Spec、计划、代码   | 任务启动 | ~10-50 KB  |
| 参考层 | 完整的治理文档、架构文档          | 按需加载 | ~50-200 KB |

**2.2 [强制]** 参考层内容不预加载，仅在需要时通过工具获取。

---

## Clause 3. 上下文加载规则

**3.1 [强制]** 会话启动时加载：

| 条款  | 内容         | 来源                             |
| ----- | ------------ | -------------------------------- |
| 3.1.1 | 角色 Persona | `.trae/agents/{role}.yaml`       |
| 3.1.2 | 项目铁律     | `.trae/rules/project_rules.md`   |
| 3.1.3 | 项目入口     | `AGENTS.md`                      |
| 3.1.4 | 知识库入口   | `docs/ai/knowledge/decisions.md` |

**3.2 [推荐]** 任务启动时按需加载：

| 条款  | 内容             | 条件               |
| ----- | ---------------- | ------------------ |
| 3.2.1 | 相关 Spec 文档   | 有已完成的规格说明 |
| 3.2.2 | 相关计划文档     | 有实施计划         |
| 3.2.3 | 相关 AI 合约     | 有关联合约         |
| 3.2.4 | GitHub Issues/PR | 涉及代码协作       |
| 3.2.5 | 记忆文件         | 需要跨会话信息     |

---

## Clause 4. 上下文自检

**4.1 [强制]** 每 15 条用户消息执行一次上下文自检：

| 条款  | 检查项                                | 动作               |
| ----- | ------------------------------------- | ------------------ |
| 4.1.1 | 当前上下文是否仍包含完成任务所需信息  | 如否，加载缺失信息 |
| 4.1.2 | 是否有不再需要的老上下文占据空间      | 移除               |
| 4.1.3 | 核心强制技能是否已加载                | 如否，强制加载     |
| 4.1.4 | 当前任务阶段是否与 Harness 工作流一致 | 如否，纠正         |

---

## Clause 5. 上下文优化建议

**5.1 [推荐]** 优化上下文的常用手段：

| 手段                     | 适用场景                      |
| ------------------------ | ----------------------------- |
| 使用 AI 合约替代完整文档 | 需要参考规则，不需要全文      |
| 分阶段加载               | 长流程任务按阶段加载          |
| 及时清理                 | 已完成任务的上下文不再保留    |
| 引用记忆                 | 用 `memory/` 替代重复描述历史 |
