## 四、告警机制

| 级别    | 条件                               | 动作               |
| ------- | ---------------------------------- | ------------------ |
| 🟢 正常 | 指标在目标范围内                   | 无                 |
| 🟡 注意 | 单指标偏离 > 20% 持续 2 周         | 产品在周报标记     |
| 🟠 警告 | 单指标偏离 > 50% 或 > 3 个同时偏离 | 启动专项改进任务   |
| 🔴 严重 | 核心指标触线（E2E/构建/漏测率）    | 冻结发布，回阶段 4 |

**核心阻塞指标**：E2E 通过率 < 100%、构建失败、漏测率 > 5%

## 六、行为调节（L2.5 自适应治理）

> 基于指标表现自动调节 Agent 行为，形成测量→反馈→矫正的闭环。

### 6.1 调节在进化引擎中的位置

```
L1 复盘度量 → L2 模式沉淀 → [L2.5 行为调节] → L3 知识更新 → L4 框架迭代
                                    ↑
                            每次周评估触发
```

### 6.2 四种调节动作

| 动作               | 级别  | 效果                             | 恢复条件          |
| ------------------ | ----- | -------------------------------- | ----------------- |
| `inject_prompt`    | L1    | 在 AGENTS.md 注入一句话纠正提示  | 下次评估自动移除  |
| `force_skill`      | L2    | 锁定某技能为强制加载             | 连续 7 天无违规   |
| `escalate_depth`   | L2-L3 | 提升该角色某阶段的执行深度       | 连续 2 周指标正常 |
| `freeze_autopilot` | L3-L4 | 冻结自主决策，所有动作需人类签署 | 人类手动解除      |

### 6.3 触发规则表（初始 8 条）

| #   | 指标               | 条件           | 动作                            | 级别 | 角色 |
| --- | ------------------ | -------------- | ------------------------------- | ---- | ---- |
| 1   | 技能遗漏率         | ≥ 3/任务 × 7天 | `force_skill`                   | L2   | 全部 |
| 2   | 需求偏差度         | ≥ 3            | `escalate_depth`(review, L1→L2) | L2   | 产品 |
| 3   | karpathy 跳过      | ≥ 3 次/周      | `force_skill`(karpathy)         | L2   | 开发 |
| 4   | 代码审查一次通过率 | < 40%          | `escalate_depth`(review, L1→L3) | L3   | 开发 |
| 5   | E2E 通过率         | < 100%         | `freeze_autopilot`              | L4   | 全部 |
| 6   | 设计可访问性阻断   | ≥ 1            | `force_skill`(accessibility)    | L2   | 设计 |
| 7   | 中文率             | < 80%          | `inject_prompt`                 | L1   | 全部 |
| 8   | 反模式出现         | ≥ 3 次         | `inject_prompt` + `force_skill` | L2   | 全部 |

### 6.4 调节存储

调节状态存储在 `.harness/adjustments.yaml`，由 `scripts/evaluate-adjustments.py` 自动管理：

- **active_adjustments**：当前生效的调节，Agent 启动时读取
- **inactive_adjustments**：已过期或已完成的调节（审计留存）

### 6.5 安全规则

1. **自动调节仅限 L1-L2**：`force_skill` 和 `inject_prompt` 自动生效
2. **L3-L4 需人类审批**：`escalate_depth`(重度) 和 `freeze_autopilot` 通过飞书表单请求批准（先审后行）
3. **所有调节 7 天过期**：过期自动移除，指标恢复后不再触发
4. **人类手动覆盖**：在 `.harness/adjustments.yaml` 中改 `status: overridden` 即停止该调节
5. **永不惩罚人类**：调节只约束 Agent 行为，不影响人类审批链路
6. **矫正块自动同步**：`.harness/adjustments.yaml` 与 AGENTS.md 中的矫正块保持同步

### 6.6 调节生命周期

```
指标采集 → 周评估 → 匹配规则 → 生成调节
                                  │
                    ┌─────────────┴─────────────┐
                    │ L1-L2                     │ L3-L4
                    ▼                           ▼
              自动生效                  飞书审批表单 → 人类确认
                    │                           │
                    └─────────────┬─────────────┘
                                  ▼
                    写入 adjustments.yaml + AGENTS.md
                                  │
                                  ▼
                    Agent 下次会话读取矫正块 → 行为受约束
                                  │
                                  ▼
                      下周评估 → 检查恢复条件
                        ├── 已恢复 → 移除调节
                        └── 未恢复 → 自动续期
```
