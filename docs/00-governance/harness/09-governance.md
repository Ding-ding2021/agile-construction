# Harness 治理

## Clause 1. 治理总则

**1.1 [强制]** Harness 治理的目的是确保项目运行在规则框架内，同时允许自适应调节。

**1.2 [参考]** 治理范围：

| 维度       | 说明                                                        |
| ---------- | ----------------------------------------------------------- |
| 结构性治理 | 文档结构、代码结构、目录规范（见 `document-governance.md`） |
| 流程性治理 | Harness 流水线执行合规性（见 `01-workflows.md`）            |
| 质量治理   | 质量标准、门禁条件（见 `project-charter.md` Clause 5）      |

---

## Clause 2. 治理指标

**2.1 [参考]** 项目关键治理指标：

| 条款  | 指标           | 阈值                         | 测量方式         |
| ----- | -------------- | ---------------------------- | ---------------- |
| 2.1.1 | 文档合规率     | ≥ 80% 文档有完整 frontmatter | 月度扫描         |
| 2.1.2 | 技能调用覆盖率 | 核心技能调用率 100%          | 15-call 自检记录 |
| 2.1.3 | 评审通过率     | 评估组全票通过率 ≥ 70%       | 评估报告统计     |
| 2.1.4 | 红线违规率     | 同模式违规 ≤ 2 次/月         | memory/ 记录     |
| 2.1.5 | 文档断裂引用率 | 断裂链接 ≤ 5 个              | 月度链接检查     |

详见 [quality-metrics.md](../quality-metrics.md)。

---

## Clause 3. 治理执行

### 3.1 自动化治理

**3.1.1 [强制]** 以下治理动作自动执行：

| 条款  | 动作         | 触发条件       | 频率     |
| ----- | ------------ | -------------- | -------- |
| 3.1.1 | 文档扫描     | 会话启动       | 每次会话 |
| 3.1.2 | 技能调用检查 | 15-call 自检点 | 每次会话 |
| 3.1.3 | 链接断裂检查 | 月度检查       | 每月     |

### 3.2 人工治理

**3.2.1 [强制]** 以下治理动作需要人类参与：

| 条款  | 动作           | 参与者     | 频率   |
| ----- | -------------- | ---------- | ------ |
| 3.2.1 | 质量指标报告   | 产品经理   | 每月   |
| 3.2.2 | 文档健康度评估 | 产品经理   | 每季度 |
| 3.2.3 | 治理规则修订   | 全角色评审 | 按需   |

---

## Clause 4. 治理违规处理

**4.1 [强制]** 治理违规分级与处理：

| 级别    | 定义                           | 处理方式                |
| ------- | ------------------------------ | ----------------------- |
| L1 轻微 | 单次技能跳过、文档格式不符     | 自动记录 + 提醒         |
| L2 中等 | 同模式违规 2 次、评估流程跳过  | 记录 + 强制加载矫正     |
| L3 严重 | 未经评估开始开发、多次红线违规 | 自动 + 人类仲裁         |
| L4 危急 | 擅自修改核心模块、绕过安全审查 | 冻结自动执行 + 人类审批 |

**4.2 [强制]** 所有违规记录在 `memory/` 中，以 `violations` 标签存储。

---

## Clause 5. 治理改进

**5.1 [推荐]** 基于治理指标数据分析，每季度提出治理改进建议。

**5.2 [推荐]** 改进建议应包含：

| 条款  | 要素                       |
| ----- | -------------------------- |
| 5.2.1 | 当前问题描述               |
| 5.2.2 | 根因分析                   |
| 5.2.3 | 改进方案（含具体条款修订） |
| 5.2.4 | 影响评估                   |
