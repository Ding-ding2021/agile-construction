GOV-027 · Design Governance

设计治理规范

定义设计 Token 的生命周期管理、组件准入门禁、主题色板治理和跨角色协作流程。确保设计体系在项目演进中保持一致性和可维护性。

版本 v1.0.0 更新 2026-05-13 状态 Active 关联 DES-001 · DES-002 · DES-004
I

治理架构

设计治理分为三个层级，逐层收敛，从原则到实现。

L1 · 原则层 DES-001 设计哲学、「最少颜色、最少装饰、组件原生、语义间距」
L2 · 规范层 DES-004 VI 规范 · GOV-027 治理规范 · 组件查询表 · 开发合约
L3 · 执行层 themes.css · 组件代码 · DES-002 检查清单 · 审查门禁

🎯 治理目标

  • 所有页面视觉一致，不依赖个人审美
  • Token 变更可追溯、有审批
  • 组件复用最大化，零重复造轮
  • 非设计师也能安全地扩展 UI

🧭 治理范围

  • 色彩系统（色板、语义色、暗色）
  • 字体系统（层级、字重、字体栈）
  • 间距与圆角 Token
  • shadcn 组件全生命周期
  • 多主题色板注册与卸除

👤 角色职责

  • 设计师 · 定义/维护 Token、审查视觉一致性
  • 开发者 · 按 Token 实现、通过门禁
  • AI Agent · 遵守合约、调用技能、不自作主张
  • 审查委员会 · 验收门禁、裁定争议
II

设计 Token 治理

所有设计 Token 的变更必须遵循标准化流程，禁止直接修改代码中的硬编码值。

色彩 Token 变更流程

1 提出变更
→
2 更新 themes.css
→
3 更新 VI 规范
→
4 视觉审查
→
5 全量回归

🎨 色彩 Token 规则 强制

  • 所有色值统一使用 oklch() 色彩空间
  • 禁止 HEX / RGB / HSL 色值进入代码
  • 语义色变量优先于直接色阶：用 text-muted-foreground 而非 text-neutral-500
  • 新增色板必须在 themes.css 同时注册亮色/暗色两套变量

📐 间距 Token 规则 强制

  • 所有间距必须为 2px 整数倍，禁止 p-2.5 m-1.5 等 0.5 步长
  • 使用 Tailwind spacing scale，不自定义间距变量
  • shadcn 组件内部间距不受此限制

🔤 字体 Token 规则 强制

  • 字体层级限制 L1-L5 五级，禁止跳级使用
  • 字重仅限 400 / 500 / 600 三个权重
  • 中英文混排必须使用 Inter + Noto Sans SC 组合
  • 字体栈变更需设计师确认

⭕ 圆角 Token 规则 强制

  • 全局基准圆角 --radius: 0.625rem（shadcn 默认）
  • 卡片圆角 rounded-lg，按钮遵循 shadcn variant
  • 禁止引入新的圆角层级

变量覆盖层级

CSS 变量按影响范围分为三层。L1 随主题色相显著变化，L2 带微弱色相提示，L3 所有主题保持一致。

L1 品牌色 色相随主题变化 · 6 个变量 --primary --accent --ring --chart-*
L2 表面色 带微弱色相提示 · 6 个变量 --bg --secondary --muted --border --input
L3 中性色 所有主题保持一致 · 12 个变量 --card --popover --destructive --sidebar

➕ 新增色板流程 强制

1 选色
→
2 编码
→
3 注册
→
4 审查

步骤：① 选择色相基础色 → ② 在 themes.css 中添加亮色/暗色变量对 → ③ 在主题选择器中注册 → ④ 全量组件视觉回归

🚫 色板准入标准 强制

  • --primary 与文本背景对比度 ≥ 4.5:1
  • 暗色模式下文字可读性通过 WCAG AA
  • 20+ 个变量必须全部覆盖（亮色 + 暗色）
  • 5 组 chart 色板内可辨识
IV

组件治理

组件复用优先级和生命周期管理，确保代码库健康演进。

复用优先级

P0 直接使用 ui/ 已有
→
P1 shadcn CLI 安装新组件
→
P2 组合现有组件
→
P3 手写自定义组件

📦 shadcn 组件管理 强制

  • 所有 shadcn 组件通过 CLI npx shadcn@latest add 安装
  • 禁止手动复制 shadcn 源码
  • 安装后必须检查自引用 bug 并修复
  • 禁止直接修改 shadcn 生成的组件代码
  • 36 个组件已安装，完整列表见组件查询表

🏗️ 自定义组件准入 推荐

  • 必须确认无现成 shadcn 组件可复用
  • 必须使用 Tailwind + shadcn 组件组合实现
  • 必须导出 TypeScript Props 接口
  • 命名导出为主，禁止默认导出
  • 注入 cn() className 支持外部样式覆盖

🔄 组件生命周期 参考

  • 调研 · 搜索 shadcn registry 确认存在性
  • 添加 · CLI 安装或组合实现
  • 验收 · 通过视觉 + 响应式 + 暗色 + a11y 门禁
  • 废弃 · 标记 deprecated 而非直接删除
  • 移除 · 确认无依赖后安全清理

🚫 禁止行为 强制

  • 禁止引入 shadcn 外的第三方 UI 库
  • 禁止内联 style={{}} prop（性能动画除外）
  • 禁止 bg-white/10 等透明度色值
  • 禁止 --pm-* 旧品牌体系变量
  • 禁止纯黑 #000 / 纯白 #FFF
V

审查门禁

每个组件或页面变更必须通过以下 4 道门禁方可合并。任何一道不通过，退回修改。

门禁 审查内容 结果
🎨 视觉一致性 色彩使用语义变量不跳级 · 字体层级合规 · 间距 2px 倍数 · 无纯黑纯白 · 图标统一 16px PASS
📱 响应式 1920 / 1360 / 1024 / 768 四断点 · 无溢出 · 表格横向滚动 · 表单字段自适应 · 移动端可操作 PASS
🌙 暗色模式 text/bg 使用语义变量 · 暗色下文字可读 · 边框色适配 · chart 色板正常 · 无 hardcode 亮色值 PASS
♿ 可访问性 颜色对比度 ≥ 4.5:1 · 按钮有 focus ring · 表单有关联 label · 图标有 aria-label · 键盘可导航 PASS

门禁不通过的典型案例

✕
用了旧品牌变量 · 页面中出现 --pm-blue、--pm-shadow-* 等旧体系变量 → 退回替换为 shadcn 语义变量
✕
暗色模式未适配 · 组件在 .dark 下文字不可读或背景色错误 → 退回使用 dark: 前缀或语义变量
✕
间距不合规 · 使用了 p-1.5、m-2.5、style={{ marginTop: 13 }} → 退回修正为 2px 倍数
✕
手写 shadcn 已有组件 · 重新实现了 Button / Input / Select → 删除自定义代码，使用 CLI 安装
