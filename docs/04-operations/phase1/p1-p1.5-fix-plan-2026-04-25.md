## 0. 修复策略总览

### 0.1 核心原则

1. **先还账，再赶路**：Wave 1+2 未完成前，不启动 Phase 2 功能开发
2. **不影响功能 = 可延后**：非阻塞项（如 MUI 封装、ProjectCard）可随 Phase 2 渐进修复
3. **改一处，验一处**：每完成一个任务立即 `npm run lint` + `npm run build`，不堆积
4. **决策优于编码**：Wave 0 的数据策略决策必须先拍板，否则 Wave 2 的 Repository 接入方向不明

### 0.2 波次划分

| 波次       | 范围     | 目标                     | 预估时间 | 是否可延后    |
| ---------- | -------- | ------------------------ | -------- | ------------- |
| **Wave 0** | 决策     | 数据层策略拍板           | 10 分钟  | ❌ 不可       |
| **Wave 1** | Hotfix   | lint 通过 + 旧代码清理   | 1~2 小时 | ❌ 不可       |
| **Wave 2** | 核心清零 | CSS 硬编码 + 导航补齐    | 1~2 天   | ❌ 不可       |
| **Wave 3** | 深度治理 | Token 压缩 + 渐变 + 组件 | 2~3 天   | ⚠️ 部分可延后 |

## 3. Wave 2：核心清零（1~2 天）

### C-1 project-detail.css 硬编码清零（核心债务，1 天）

**问题**: `project-detail.css` 是当前最大单笔 CSS 债务 — 27 处 HEX + 69 处 rgba = **96 处硬编码**。

**执行策略**: 分批替换，从高频到低频

**批次 1 — 背景色/边框色（高影响，1 小时）**:

```css
/* 常见模式：rgba(255,255,255,0.05) → 暗色表面 */
--pm-surface-50: rgba(255, 255, 255, 0.05);

/* 常见模式：rgba(255,255,255,0.08) → 暗色表面 hover */
--pm-surface-80: rgba(255, 255, 255, 0.08);

/* 常见模式：rgba(255,255,255,0.1) → 暗色边框 */
--pm-border-light: rgba(255, 255, 255, 0.1);

/* 常见模式：rgba(0,0,0,0.2) → 深色阴影/遮罩 */
--pm-shadow-deep: rgba(0, 0, 0, 0.2);

/* 常见模式：#ffffff → 主文字 */
--pm-text-primary: #ffffff;

/* 常见模式：#e2e8f0 → 次要文字 */
--pm-text-secondary: #e2e8f0;
```

**批次 2 — 功能色（中影响，2 小时）**:

```css
/* 状态色：success/warning/error/info — 检查是否已有对应变量 */
--pm-status-success: #52c41a;
--pm-status-warning: #faad14;
--pm-status-error: #f5222d;
--pm-status-info: #1890ff;
```

**批次 3 — 一次性色值（低影响，2 小时）**:

- 扫描 `#` 后 6 位不常见的色值（如 `#3b82f6`、`#8b5cf6` 等）
- 判断其语义（品牌主色？辅助色？标签色？）
- 映射到已有变量或新建语义变量

**工具命令**:

```bash
# 提取所有 HEX
grep -oP '#[0-9a-fA-F]{3,6}' src/components/project/project-detail.css | sort | uniq -c | sort -rn

# 提取所有 rgba
grep -oP 'rgba\([^)]+\)' src/components/project/project-detail.css | sort | uniq -c | sort -rn
```

**验收标准**:

- [ ] `grep "#\|rgba(" src/components/project/project-detail.css` 返回 ≤ 20 处（保留 MUI 覆盖和复杂渐变）
- [ ] 项目详情页视觉回归通过（与原设计一致，无颜色跳变）
- [ ] `npm run build` 通过

## 4. Wave 3：深度治理（2~3 天，部分可延后）

### G-1 CSS Token 压缩到 80 个以内（1 天，⚠️ 可延后）

**问题**: `index.css` 有 148+ 个变量，命名混乱，无分层。

**目标结构**（三层 Token）:

```css
/* Layer 1: 基础 Token（色值/字号/间距 — 不随主题变） */
:root {
  --pm-base-white: #ffffff;
  --pm-base-black: #000000;
  --pm-base-gray-50: #f9fafb;
  /* ... 最多 20 个 */
}

/* Layer 2: 语义 Token（背景/边框/文字 — 随主题变） */
:root {
  --pm-bg-primary: var(--pm-base-white);
  --pm-bg-surface: rgba(255, 255, 255, 0.05);
  --pm-text-primary: var(--pm-base-white);
  --pm-border-default: rgba(255, 255, 255, 0.1);
  /* ... 最多 40 个 */
}

/* Layer 3: 组件 Token（卡片/按钮/表格 — 由语义 Token 派生） */
:root {
  --pm-card-bg: var(--pm-bg-surface);
  --pm-card-border: var(--pm-border-default);
  --pm-btn-primary-bg: var(--pm-color-primary);
  /* ... 最多 20 个 */
}
```

**执行策略**: 不一次性全改，先建立新命名规范，然后逐个文件迁移。

- 新建 `src/styles/tokens.css`，按三层结构定义
- `index.css` 保留旧变量作为兼容层（加 `@deprecated` 注释）
- 新页面/组件强制使用新 Token；旧页面修改时顺手替换

**验收标准**:

- [ ] 新 Token 文件 `tokens.css` 变量 ≤ 80 个
- [ ] 新页面全部使用新 Token
- [ ] 旧页面迁移比例 ≥ 30%
