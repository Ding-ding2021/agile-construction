# 需求阶段

## 目标

明确产品要解决的问题、目标用户、核心功能和成功指标。

## 关键活动

### 1. 需求收集

- **用户访谈**：与目标用户进行1对1深度访谈
- **竞品分析**：分析同类产品功能与体验
- **数据分析**：通过埋点数据分析用户行为
- **业务调研**：了解业务流程与痛点

### 2. 需求分析

- **用户画像**：定义目标用户特征
- **场景梳理**：列出用户核心使用场景
- **痛点排序**：按影响程度和频率排序
- **价值评估**：评估每个需求的业务价值

### 3. PRD编写

```markdown
## PRD标准结构

1. 背景与目标
   - 项目背景
   - 业务目标
   - 成功指标

2. 需求范围
   - In Scope
   - Out of Scope

3. 用户故事
   - 作为 [角色]
   - 我希望 [功能]
   - 以便 [价值]

4. 功能详述
   - 功能列表
   - 业务流程图
   - 页面原型

5. 非功能需求
   - 性能要求
   - 兼容性要求
   - 安全要求

6. 附录
   - 术语表
   - 参考文档
```

### 4. 需求评审

- **评审参会**：PM、设计、开发、测试、业务方
- **评审内容**：完整性、可行性、一致性
- **评审产出**：评审纪要、待确认事项清单

## 准入/准出标准

| 检查项   | 准入标准             | 准出标准                 |
| -------- | -------------------- | ------------------------ |
| 需求来源 | 业务方提出或产品规划 | 已记录并分类             |
| PRD文档  | 无                   | 完成并通过评审           |
| 原型图   | 无                   | 核心流程有低保真原型     |
| 技术评估 | 无                   | 开发Leader完成工作量评估 |

## 常见问题

1. **需求频繁变更** → 建立变更控制流程，评估影响
2. **需求理解不一致** → 用用户故事+验收标准明确
3. **技术可行性盲区** → 早期引入技术预研
