## 三、专项需求文档（P0 部分）

### 需求 1：无限日历加载

**参照**：ClickUp、GanttPRO 在缩小视图时会自动加载更宽的时间范围，日历 API 被调用获取新范围的非工作日数据。

**现状**：`range` 计算依赖 `rawRange.end`（WBS 数据的最晚日期），缩小时范围被截断。

**方案**：
1. `range.end` 完全脱离 WBS 数据，纯 `dayWidth` 驱动
2. 生成的天数足够覆盖最小画布宽度（如 `totalDays = max(365, 2000 / dayWidth)`）
3. `useEffect` 中 `range.start` → `range.end` 变更时触发 `loadNonWorkingDays`
4. 连续缩放中忽略过期 API 响应（`abortRef`）

### 需求 2：里程碑

**参照**：GanttPRO 的菱形标记 ◆，无工期，只标记一个日期点。

**实现**：
1. WBSNode 新增 `nodeLevel: 'milestone'` 枚举值
2. 甘特条在 `isMilestone` 时呈现为菱形 ◆（宽度 = dayWidth，定位在 `dayStart * dayWidth`）
3. 菱形可用纯 CSS `transform: rotate(45deg)` + 小方块实现

### 需求 3：依赖类型

**参照**：ClickUp/GanttPRO 支持 FS/FF/SS/SF 四种依赖，含 Lag/Lead。

**实现**：
1. WBSNode.dependencies 字段格式增强：`"depId:type:lag"`，如 `"3:FS:2"` 表示 id=3 Finish-to-Start 提前 2 天
2. 甘特图依赖箭头根据不同类型（FS/FF/SS/SF）调整起始/终点锚点位置
3. 自动重排程：FS 依赖中前序任务推迟 → 后序自动移动
