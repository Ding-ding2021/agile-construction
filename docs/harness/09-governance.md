# 治理与考核

## 概述

51 项指标 × 五级考核节奏 × 飞书通知管道。产品是治理的主责人。

---

## 一、考核节奏

| 级别   | 时间           | 触发方式             | 负责人 | 产出                                                  |
| ------ | -------------- | -------------------- | ------ | ----------------------------------------------------- |
| 每任务 | 提交后立即     | post-commit hook     | 自动化 | `.workbuddy/stats/YYYY-MM-DD.json`                    |
| 每日   | 周一至五 18:00 | cron: `0 18 * * 1-5` | 产品   | `.workbuddy/stats/daily/YYYY-MM-DD.md` + 飞书         |
| 每周   | 周五 17:00     | cron: `0 17 * * 5`   | 产品   | `.workbuddy/stats/weekly/YYYY-Www.json` + 周报 + 飞书 |
| 每月   | 1 号 09:00     | cron: `0 9 1 * *`    | 产品   | `.workbuddy/stats/trends/YYYY-Qq.json` + 月报 + 飞书  |
| 每发布 | tag push       | `on.tags`            | 产品   | `.workbuddy/stats/release/vX.Y.Z.json`                |

## 四、告警机制

| 级别    | 条件                               | 动作               |
| ------- | ---------------------------------- | ------------------ |
| 🟢 正常 | 指标在目标范围内                   | 无                 |
| 🟡 注意 | 单指标偏离 > 20% 持续 2 周         | 产品在周报标记     |
| 🟠 警告 | 单指标偏离 > 50% 或 > 3 个同时偏离 | 启动专项改进任务   |
| 🔴 严重 | 核心指标触线（E2E/构建/漏测率）    | 冻结发布，回阶段 4 |

**核心阻塞指标**：E2E 通过率 < 100%、构建失败、漏测率 > 5%

```markdown

## 告警

- 无 / 🟡 ... / 🟠 ... / 🔴 ...

---

发布者：产品
发布渠道：.workbuddy/stats/daily/YYYY-MM-DD.md
通知：飞书 webhook
```

## 六、飞书通知

### 路径 A：Webhook（当前方案）

1. 飞书群 → 群机器人 → 添加自定义机器人 → 复制 webhook URL
2. 存入 GitHub Secrets: `FEISHU_WEBHOOK_URL`
3. GitHub Actions: `curl -X POST $FEISHU_WEBHOOK_URL -d @report.json`

### 路径 B：lark-cli（进阶方案，待授权）

当前状态：lark-cli `NOT_CONFIGURED`。需飞书开放平台创建企业自建应用后 OAuth 授权。
