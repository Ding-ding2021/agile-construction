# UI 组件开发检查清单

**版本**: v1.2.0  
**更新日期**: 2026-05-02

## 一、开发前准备

### 1.1 设计稿对齐

- [ ] 已查看 Pixso 设计稿 (路径: Pixso 设计文件或 `design_to_code` MCP 工具)
- [ ] 已截取设计稿截图对比 (`get_image` MCP 工具)
- [ ] 已标注关键尺寸: 宽度、高度、间距、圆角、字体大小
- [ ] 已确认组件在页面中的位置和布局
- [ ] 已确认组件的业务优先级和展示位置

### 1.2 设计系统对齐

- [ ] 确认使用的颜色变量 (查看 `src/index.css` 中的 `--pm-*` 变量)
- [ ] 确认使用的字体大小 (参考设计规范字体系统)
- [ ] 确认使用的圆角规范 (8px / 10px / 14px / 16px / 999px)
- [ ] 确认使用的阴影效果 (查看 `--pm-shadow-*` 变量)
- [ ] 确认使用的间距规范 (8px / 12px / 16px / 21px / 24px)

### 1.3 业务逻辑确认

- [ ] 已理解组件的业务价值
- [ ] 已确认组件的数据来源
- [ ] 已确认组件的交互逻辑
- [ ] 已确认组件的优先级和展示位置
- [ ] 已确认组件与其他组件的关系

## 二、开发过程

### 2.1 布局验证

- [ ] 组件宽度符合设计规范
- [ ] 组件高度自适应或符合设计
- [ ] 内间距符合规范 (8px / 12px / 16px / 21px / 24px)
- [ ] 外边距符合规范 (8px / 12px / 16px / 24px / 32px)
- [ ] 使用 CSS 变量而非硬编码值
- [ ] 没有破坏原有布局

### 2.2 样式规范

- [ ] 使用标准卡片类名 `.card`
- [ ] 卡片内间距使用 `var(--pm-spacing-lg)` (21px)
- [ ] 卡片标题使用 `.card-header h2`
- [ ] 使用设计系统定义的颜色变量
- [ ] 圆角统一: 16px (卡片) / 14px (按钮) / 10px (子元素) / 8px (小元素)
- [ ] 没有使用未定义的颜色值
- [ ] 没有使用未定义的间距值
- [ ] 没有使用未定义的圆角值

### 2.3 字体规范

- [ ] 标题使用正确的字体大小 (14px / 16px / 18px / 24px)
- [ ] 正文使用正确的字体大小 (12px / 14px)
- [ ] 辅助文字使用正确的字体大小 (10px / 12px)
- [ ] 字重符合规范 (400 / 500 / 600 / 700)
- [ ] 行高符合规范

### 2.4 响应式验证

- [ ] 在 1920px 宽度下验证布局
- [ ] 在 1360px 宽度下验证布局
- [ ] 在 1024px 宽度下验证布局
- [ ] 在 768px 宽度下验证布局
- [ ] 组件在不同屏幕尺寸下没有溢出
- [ ] 组件在不同屏幕尺寸下布局合理

## 四、常见问题检查

### 4.1 布局问题

**问题**: 组件宽度不符合预期

检查项:

- [ ] 父容器的 `display` 属性是否正确
- [ ] `flex` 或 `grid` 布局配置是否正确
- [ ] `max-width` 和 `min-width` 设置是否合理
- [ ] 是否破坏了原有布局

### 4.2 样式问题

**问题**: 样式不一致

检查项:

- [ ] 是否使用了设计系统的 CSS 变量
- [ ] 是否有硬编码的颜色值
- [ ] 是否有硬编码的间距值
- [ ] 是否有硬编码的圆角值
- [ ] 是否有内联样式

### 4.3 响应式问题

**问题**: 某些屏幕尺寸下显示异常

检查项:

- [ ] 是否有媒体查询适配
- [ ] 是否使用了 `minmax()` 定义弹性尺寸
- [ ] 是否使用了 `clamp()` 定义范围值
- [ ] 是否在多个断点测试过
