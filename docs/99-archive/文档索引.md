# 文档索引（设计规范治理）

> **版本**: v1.0.0  
> **更新日期**: 2026-04-16  
> **目标**: 明确文档分层、单一事实源与维护责任，避免规范漂移。

---

## 1. 分层与角色

| 层级 | 文档                                         | 角色                 | 状态            | 是否执行依据 |
| ---- | -------------------------------------------- | -------------------- | --------------- | ------------ |
| L1   | `DESIGN_SPECIFICATION.md`                    | 设计总览/理念导航    | Active-Overview | 否           |
| L2   | `docs/00-governance/design-specification.md` | 唯一执行规范（SSOT） | Active          | 是           |
| L3   | `docs/00-governance/design-checklist.md`     | 开发与验收检查清单   | Active          | 依赖 L2      |

---

## 2. 维护信息

| 文档                                         | 用途                           | 负责人                  | 更新频率             | 依赖关系                         |
| -------------------------------------------- | ------------------------------ | ----------------------- | -------------------- | -------------------------------- |
| `DESIGN_SPECIFICATION.md`                    | 设计理念、模式说明、总览导航   | 设计负责人 + 前端负责人 | 每月或重大视觉调整时 | 指向 L2                          |
| `docs/00-governance/design-specification.md` | 颜色 Token、组件规范、实现口径 | 前端负责人              | 每次样式体系变更时   | 校验 `src/index.css` 的 `--pm-*` |
| `docs/00-governance/design-checklist.md`     | 研发落地检查项、验收口径       | 前端负责人 + QA         | 每次规范变更后同步   | 依赖 L2                          |

---

## 3. 引用规则（强制）

- 所有“设计规范”引用统一指向：`docs/00-governance/design-specification.md`
- 根目录 `DESIGN_SPECIFICATION.md` 仅作总览，不承载执行细则
- 新增文档若引用设计规范，必须在 PR 中注明是否影响 L2/L3
