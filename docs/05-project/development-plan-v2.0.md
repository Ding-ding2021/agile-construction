# 连锁门店营建管理系统 - 详细开发计划 V2.1

> **文档版本**: V2.1
> **适用场景**: 单人 + AI 编码模式（基于 shadcn/ui + `src-next/` 新栈）
> **计划性质**: 工程任务分解（按依赖关系排序）
> **当前基线**: Phase 1 ~ 5 已完成，Phase 6 Agent 待开发
> **核心原则**: 状态机驱动；标准先行；任务原子化；模块级测试覆盖

---

## 1. Phase 2：WBS & 标准 & 任务数据硬化

> **状态**: ✅ 已完成  
> **核心产出**：WBS 三视图（树/甘特图/网络图）+ 项目详情 8 标签页 + 标准管理三层结构 + 任务数据关系真实化

---

### P2-T1 WBS 阶段 2 — 甘特图 + 网络图 ✅

- **目标**：在阶段 1 树视图基础上，实现甘特图和网络图视图，完成 WBS 三视图。
- **完成情况**：
  1. **甘特图实现**：基于 `@svar-ui/react-gantt`，支持日/周/月/季/年粒度切换、进度条、今天标记线、周末着色
  2. **PRO 功能实现**：关键路径（CPM 算法）/ 工作日日历（dayjs business-days）— 已完成。基线对比/撤销重做（Zustand temporal）— 待实现
  3. **网络图实现**：基于 `@xyflow/react`（React Flow）+ dagre 自动布局，PERT 依赖图 ✅
  4. **三视图共享数据**：共用 WBS Store（Zustand），切换无同步成本 ✅
  5. **跨模块复用设计**：网络图同时为审批流程设计器、质检流程编排预留节点图基础设施
- **拖拽交互说明**：甘特图目前为"查看模式"——可以浏览、切换视图，但拖拽调整日期/工期、拖拽创建依赖箭头尚未实现，归属未来阶段
- **验收结果**：
  - 甘特图：进度条 / 今天线 / 日周月季年粒度切换 ✅ | 日期拖拽 / 依赖连线 ❌
  - 网络图：dagre 自动布局 / 节点状态色标 / 依赖线高亮 ✅
  - 三视图数据一致，切换无闪烁 ✅
  - 单元测试覆盖 CPM 算法、进度汇总 ✅
  - E2E 测试覆盖核心交互（展开、编辑） ✅

  2. **范围与任务**：WBS 树视图嵌入（复用阶段 1/2 产出）、范围基准、标准绑定展示 ✅
  3. **进度管理**：甘特图嵌入（复用 P2-T1）、里程碑列表、计划 vs 实际对比 ✅
  4. **成本与采购**：采购申请摘要、订单列表、预算 vs 实际 ✅
  5. **质量与验收**：验收申请状态、检查项完成度、整改任务摘要 ✅
  6. **资源与人员**：项目成员列表、工队分配入口 ✅
  7. **风险与沟通**：风险登记册、问题日志 ✅

  - 8 个标签可正常切换，URL 同步更新 ✅
  - 每个标签有独立加载状态和空态 ✅
  - 范围与进度标签嵌入真实 WBS 视图 ✅
  - 数据来自 API 非 mock ✅
  - `npm run build` 零报错 ✅

### P2-T3 标准管理基础 ✅

- **目标**：建立标准库完整体系 — 标准来源、条款结构化、规则项引擎。
- **完成情况**：
  1. **后端实体**：`standards` 表 + `standard_clauses` 表 + `standard_rules` 表 + RESTful API ✅
  2. **条款结构化**：条款编号、标题、内容、类型（执行/验收/通用）✅
  3. **规则项引擎**：`standard_rules` 表，关联条款，判定方式（是/否、数值范围、枚举）✅
  4. **前端页面**：`StandardManagementPage` 重构为 shadcn，标准列表 + 详情 + 条款管理 ✅
  5. **标准来源类型**：品牌标准、行业规范、供应商要求、项目补充 ✅
  6. **标准绑定**：任务模板可绑定标准条款，生成快照 ✅
- **验收结果**：
  - 可创建/编辑/删除标准来源 ✅
  - 标准可拆分为结构化条款 ✅
  - 条款可细化为规则项 ✅
  - 标准列表可按品牌、店型、来源类型筛选 ✅
  - 规则项可与任务模板绑定 ✅
  - AI 辅助条款提取 — 待实现

---

### P2-T4 任务数据硬化 ✅

- **目标**：将任务模块从"演示级 mock"升级为有真实数据结构、状态机守卫、父子关系的可独立工作单元。
- **完成情况**：
  1. **字段精简与硬化**：任务字段精简至核心字段，日期/金额/数量字段改为数值类型 ✅
  2. **父子关系真实化**：`parent_task_id` 自引用外键 + `tree_level`/`tree_path`（Materialized Path）✅
  3. **标准绑定真实化**：`executionStandards`/`acceptanceStandards` 从 `string[]` → `standardClauseIds[]` ✅
  4. **状态机守卫真实化**：守卫条件从 mock → 数据库查询 ✅
  5. **任务关系建模**：`task_relations` 表（前置依赖）+ 整改派生 + 派单流程 ✅
  6. **检查项闭环**：检查项独立实体化，录入/判定/整改自动派生 ✅
  7. **API 完善**：CRUD + 子树查询 + 依赖查询 + 批量创建 ✅
- **验收结果**：
  - 任务父子关系通过 `parent_task_id` 真实关联 ✅
  - 6 状态流转 + 守卫条件全部真实（非 mock）✅
  - 前置依赖可创建/查询/影响流转 ✅
  - 验收不通过 → 整改任务自动派生 ✅
  - 检查项独立实体化，录入/判定/整改闭环 ✅
  - 单元测试覆盖全部守卫和流转路径 ✅

## 5. 质量门禁

1. `npm run build` 零报错
