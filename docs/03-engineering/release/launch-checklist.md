## 阶段3启动总清单（方案B）

### 1. 目标与边界

- **目标**：把阶段2“可演示闭环”升级为“可运营闭环”，优先恢复云端鉴权并完成五条主链真链路回归。
- **执行架构**：CloudBase 快速落地（方案B）。
- **边界**：阶段3不引入新框架，不做大规模 UI 翻新；保留本地兜底，但必须远端异常可见化。

### 2. 周里程碑（你 + AI）

- **W1（4/13-4/19）**：恢复鉴权与主链真回归
  - 交付：鉴权恢复记录、五接口回归记录、阻断项清单
  - DoD：`/projects/state`、`/tasks/state`、`/acceptance/state`、`/settlement/state`、`/audit/logs` 均至少1次真实通过
- **W2（4/20-4/26）**：稳定性与一致性加固
  - 交付：错误语义对齐、幂等策略统一、回退可见化上线
  - DoD：远端失败可见提示生效；重复写入无脏数据；异常可定位
- **W3（4/27-5/3）**：阶段3业务后端推进
  - 交付：客服/工单 Agent、结算对账 Agent 的后端联调入口
  - DoD：至少一条业务流可演示且可复测

### 3. 依赖顺序（必须按序）

1. 鉴权恢复
2. 五主链真回归
3. 远端降级可见化
4. 周指标与评审口径统一
5. 周节奏发布与追踪

### 4. 风险与阻断闭环

- **鉴权阻断**：token 失效 / 环境配置错误
  - 处理：先恢复 `envId + token`，失败时保留本地兜底并记录阻断
- **接口语义不一致**：错误码、重试、幂等冲突
  - 处理：统一错误分类，固定 `X-Idempotency-Key` 规则
- **静默降级风险**：用户无感知
  - 处理：统一消费 `pm:remote-fallback` 并弹出可见提示

### 5. 周评审证据包模板

- **本周目标达成**：完成 / 未完成（原因）
- **回归结果**：接口通过率、失败样例、修复建议
- **风险状态**：新增风险、存量风险、清除风险
- **下周入口**：明确首日可执行动作（无需口头补充）

### 6. 最小验收标准（阶段3启动）

- 五主链接口真回归通过并有证据
- 降级与异常用户可见
- 周评审包可独立阅读
- 你+AI协作节奏可持续运行
