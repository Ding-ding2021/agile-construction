# 不可违背规则

> 从 AGENTS.md + document-governance.md + MEMORY.md 提取的最强约束。
> AI Agent 在每次会话中必须遵守。由精炼任务维护。

## 规则索引

| 来源 | 类别 | 规则数 |
| ---- | ---- | ------ |

---

## 语言规则

1. **全部中文**：每一句话都必须用中文，代码、路径、技术术语除外
2. **10 轮自检**：每 10 轮对话结束后自检语言，发现英文立刻纠正
3. **角色名**：产品经理叫林墨

---

## 技能规则

1. **意图匹配**：每次收到新用户意图，必须查 `docs/00-governance/harness/03-skills.md` 映射表
2. **强制加载**：有匹配 → 必须调 `skill` 工具加载，加载后声明"已加载 <技能名>"
3. **显式声明**：无匹配 → 声明"未匹配对应 skill，直接执行"

---

## 文档规则

1. **文档变更必同步**：任何 `docs/` 下文档变更后 → 必须加载 `document-sync` 技能
2. **合约自检**：AI 合约 ≤ 200 行、零叙事段落、frontmatter 完整、双向链接有效
3. **索引入口唯一**：`docs/README.md` 为唯一文档导航入口
4. **状态模型**：draft（禁止作为执行依据）→ active（唯一事实源）→ superseded → archived
5. **同主题单 active**：同主题仅允许一个 active 文档
6. **根目录限制**：仅允许 AGENTS.md、README.md 两个入口文件
7. **SSOT 原则**：每类信息只有一个唯一源，禁止重复存放

---

## 编码规则

1. **编码前加载技能**：任何代码修改前 → 必须调用 `skill` 工具加载 `karpathy-guidelines`
2. **完成前验证**：声称完成前必须 `npm run lint` → `npm run build`
3. **领域层保护**：`src/domain/`、`src/data/` 层禁止 AI 直接修改
4. **核心模块三输出**：完整单元测试 + 文本流程图 + 边界条件清单
5. **一次一个任务**：每次只改一个任务

---

## 完成规则

1. **task-memory**：任务结束后必须写入每日工作日志
2. **agentmemory_save**：关键架构决策和用户偏好必须保存
3. **三关验证**：lint / build / test:e2e（Markdown 任务跳过）

---

## 进化规则

1. **启动自检**：会话启动时检查 `docs/ai/context/state.md` 的 last_distill 时间
2. **阈值提炼**：自上次提炼后累积 ≥ 5 篇新日志 → 先执行提炼再处理用户任务
3. **定期精炼**：上次 Lint > 7 天 → 先执行精炼再处理用户任务

---

## 反模式（从自检中提取）

以下是从会话中提取的重复错误，已发生 ≥ 2 次，作为硬性禁止规则：

| #   | 反模式                                   | 出现次数 | 替代做法                                 |
| --- | ---------------------------------------- | -------- | ---------------------------------------- |
| 1   | 思考过程使用英文                         | 3+       | 必须全程中文思考，代码/路径/术语除外     |
| 2   | 收到意图后不加载对应 skill 直接行动      | 2+       | 先查技能铁律映射表，匹配则必须加载 skill |
| 3   | 不先问需求就跳入实现                     | 2+       | 必须先澄清需求和意图，再规划，再执行     |
| 4   | 存在设计文档/旧 session 记录却重复问用户 | 2+       | 先查已有文档和 agentmemory，再问剩余问题 |
| 5   | 写过规则自己不遵守（自检/技能/验证）     | 2+       | 规则制定后必须写 self-check 步骤         |
