# AI 合约：V1 产品定义

## 模块定位

L2 产品定义文档，定义 V1 交付范围、三大核心能力详解、模块依赖关系和交付标准。

## 三大核心能力

| 能力           | 核心假设                                     | 交付判定                                       |
| -------------- | -------------------------------------------- | ---------------------------------------------- |
| 原子化项目管理 | 项目可拆解为原子任务，项目状态从任务聚合推导 | 任务独立推进、项目状态实时聚合、多品牌模板适配 |
| 标准驱动执行   | 标准库结构化，任务从模板自动生成             | 条款级管理、模板匹配生成、标准关联执行         |
| AI 赋能降本    | Agent 辅助降低专业门槛                       | 至少一个场景可用、人工确认/驳回、全程留痕      |

## 模块清单（V1）

| 模块     | 优先级 | 定位                     |
| -------- | ------ | ------------------------ |
| 项目管理 | P0     | 项目容器、概览、配置     |
| 任务中心 | P0     | 任务树、状态机、执行推进 |
| 人员管理 | P0     | 人员、工队、角色权限     |
| 标准管理 | P0     | 标准库、模板中心         |
| 采购管理 | P1     | 采购申请、订单管理       |
| AI Agent | P1     | 辅助建议                 |
| 验收管理 | P1     | 并入项目/任务中心        |
| 系统设置 | P0     | 权限、角色、配置         |

## 模块依赖关系

- 项目管理 → 任务中心（状态聚合）
- 任务中心 → 标准管理（模板生成）
- 任务中心 → 人员管理（任务分配）
- 项目管理/任务中心 → 采购管理（采购需求）
- 项目管理/任务中心 → AI Agent（辅助建议）
- 系统设置 → 权限控制

## V1 角色

- 营建主管（统筹项目、配置模板、分配任务）
- 营建专员（推进任务、发起采购、验收）
- 工队（接收任务、提交进度）
- 系统管理员（配置维护）

## V1 不做什么

- 平台化/多方协同
- 商业化/计费
- 开放 API / 第三方集成
- 移动端原生应用
- 多语言/国际化
- 实时协作/多人编辑

## 质量门禁

1. 任务状态机无非法跳转（单元测试覆盖）
2. 项目聚合状态与任务实际状态一致
3. Agent 建议可审计可追踪
4. 权限配置生效
5. P0 模块全部完成
6. 一条完整端到端业务流跑通
