# AI 合约：人机协作双轨策略

## 模块定位

定义人类（决策者）与 AI（执行者）在项目中的双轨协作流程，以及三层记忆架构的管理规范。

## 核心实体

| 实体     | 字段                         | 状态机                                               |
| -------- | ---------------------------- | ---------------------------------------------------- |
| 人类决策 | 需求描述、反馈内容、审批结果 | 提出需求 → 阅读文档 → 反馈 → 批准/驳回               |
| AI 执行  | 文档输出、角色委派、评审发起 | 澄清需求 → 产出规范 → 委派角色 → 组织评审 → 交付汇报 |
| 记忆条目 | 层级(L1/L2/L3)、内容、时间戳 | 临时 → 短期 `memory/` → 长期 MCP Memory              |

## 业务规则

1. AI 负责编写 `docs/` 下所有文档，人类只阅读和审批，不参与编写
2. 协作流水线：人类需求 → AI 写 docs → 人类阅读 → 人类反馈 → AI 修改 → 人类终审 → document-sync
3. `docs/` 是 SSOT，`memory/` 和 MCP Memory 是缓存
4. AI 发现的重要模式必须先写入 `memory/`，再通过汇报由人类决定是否升级到 `docs/`

- 每次交付必须更新 `memory/progress-log.md`
- 新会话启动时，AI 必须先读取 `memory/current-task.md`
