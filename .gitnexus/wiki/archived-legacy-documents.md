# 归档与遗留文档模块

## 概述

**归档与遗留文档**模块（`docs/99-archive/`）作为连锁建店管理系统的历史记录和设计基础。它包含了已被取代的设计规范、产品规划文档和架构蓝图，记录了系统从概念到V1 MVP实施阶段的演进过程。

该模块为**只读参考资料**——它记录了影响当前代码库的设计决策、建模方法和产品策略，但不再进行主动维护。

## 模块内容

### 1. 设计规范（v2.0 原始版）

**文件：** `DESIGN_SPECIFICATION_v2.0_original.md`

这是Liquid Glass + Material Design 3 UI框架的完整设计系统规范。它定义了：

- **设计系统基础：** 色彩系统（品牌蓝 `#154DD9`、语义色、玻璃效果）、排版（字体族、字号、字重）、间距系统（4px基本单位）、圆角和阴影系统
- **组件库：** 按钮变体（主要、次要、破坏性、幽灵、图标）、卡片类型（统计、项目、看板）、表单组件（输入框、搜索框、文本域、选择器）、徽章和标签、进度条
- **页面规范：** 布局结构、主要页面模板、统计卡片模块、工具栏设计、视图模式（网格/列表/看板/地图/日历）、数据筛选和分页
- **交互与动画：** 悬停状态、过渡（200ms标准）、缩放效果、加载动画
- **实施指南：** 组件使用规则、代码示例、合规检查清单、页面合规性追踪

**关键设计令牌：**

```css
--brand-primary: #154dd9;
--bg-primary: #030b1a;
--bg-secondary: #0a2363;
--glass-bg-light: rgba(255, 255, 255, 0.04);
--text-primary: rgba(255, 255, 255, 1);

```

### 2. 产品规划文档（遗留中文版）

**文件：** `legacy-chinese/产品规划文档.md`

本文档定义了多智能体建店管理平台的V1/MVP产品策略。它涵盖了：

- **系统定位：** 面向品牌运营部门的标准化建店管理系统
- **核心价值主张：** "把专业施工能力交给系统，把品牌精力留给品牌"
- **业务领域：** 项目管理、标准管理、采购管理、资产管理
- **核心建模：** 任务树模型（任务为最小执行单元）、项目/流程/服务统一视图、结果对象模型
- **模块优先级：** P0（工作台、项目管理、任务中心、标准管理、采购、验收、资产管理、人员、系统设置）、P1（合同结算、数据中心、工单）、P2（资源门户、平台商业化）
- **版本路线图：** V1.0 MVP → V1.5 增强版 → V2.0 平台版
- **智能体规划：** 7个业务智能体（品牌需求、项目经理、调度、验收/质检、结算、客服）

### 3. 任务中心需求文档（遗留中文版）

**文件：** `legacy-chinese/任务中心需求文档.md`

任务中心模块的详细需求——系统的执行主干。涵盖了：

- **模块定位：** 连接项目容器、标准库、资源调度、验收和结果对象的执行主线控制台
- **业务对象：** 任务、父任务、子任务、任务关系、执行检查清单、任务提交、任务提醒、阻塞/衍生任务记录
- **任务结构：** 4层层级结构（项目根节点 → 阶段 → 工作包 → 执行任务）、继承规则、依赖管理
- **标准绑定：** 执行标准（如何做）、验收标准（如何通过）、标准快照
- **状态机：** 9个状态（待创建 → 待分配 → 待执行 → 执行中 → 待提交 → 待验收 → 失败 → 已完成 → 已关闭）
- **页面结构：** 任务中心首页、调度池、任务详情页、关系视图、日历视图
- **权限：** 品牌隔离、项目范围隔离、资源提供方隔离、最小权限原则

### 4. 任务树建模说明（遗留中文版）

**文件：** `legacy-chinese/任务树建模说明.md`

任务树模型的架构规范——系统的核心执行模型。定义了：

- **建模原则：** 原子性、层级性、标准化、状态驱动、结果分离、可审计性、可控复杂度
- **核心对象：** 任务、父任务、子任务、任务关系（父子、依赖、衍生、关联）、任务模板、标准快照
- **层级模型：** 单父树结构 + 任务关系扩展（不混合树与网络关系）
- **状态模型：** 9个标准状态，每个转换都有守卫条件
- **标准绑定：** 4个绑定级别（模板 → 任务生成 → 任务创建 → 执行快照）
- **自动生成逻辑：** 模板 + 规则 + 智能体建议 + 人工确认
- **智能体协作：** 仅限受限节点能力，无无边界决策
- **V1边界：** 仅限单个项目、单个父任务、无跨项目共享、无循环依赖

### 5. V1 PRD（遗留中文版）

**文件：** `legacy-chinese/多Agent建店管理平台_V1_PRD.md`

V1 MVP的产品需求文档。定义了：

- **项目定义：** 多智能体建店管理平台，用7个智能体替代标准化人工岗位
- **系统范围：** 品牌门户、资源调度平台、资源提供方门户
- **智能体角色：** 客服（常见问题解答、工单创建）、品牌需求（结构化录入）、项目经理（自动推进）、调度（资源匹配）、执行（任务指导）、验收（质量检查）、结算（对账）
- **核心业务流程：** 需求录入 → 项目创建 → 任务分解 → 调度 → 执行 → 验收 → 结算
- **状态机：** 项目（10个状态）、任务（8个状态）、工单（4个状态）、验收（6个状态）、结算（5个状态）
- **成功指标：** 40%+自动回复率、60%+自动完成率、30%+自动调度覆盖率、50%+自动检查覆盖率
- **异常处理：** 10+种需要人工干预的情况

### 6. V1 技术方案（遗留中文版）

**文件：** `legacy-chinese/多Agent建店管理平台_V1_技术方案.md`

V1的技术架构草案。涵盖了：

- **架构层次：** 前端应用 → API网关 → 业务服务 → 智能体编排层 → 数据层 → 外部能力
- **设计原则：** 业务状态与智能体状态分离、流程驱动（非智能体自由形式）、所有智能体操作可审计、规则可配置
- **系统拆分：** V1推荐模块化单体架构（认证、项目、任务、调度、资源、验收、结算、工单、智能体模块）
- **智能体运行时：** 注册中心、提示词模板中心
