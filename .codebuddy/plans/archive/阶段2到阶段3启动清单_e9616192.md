## User Requirements

- 基于已完成的阶段2成果，产出“阶段2→阶段3启动清单”，用于直接推进下一阶段执行。
- 清单需明确：优先级、依赖顺序、周里程碑、责任分配、交付物、验收口径。

## Product Overview

- 当前系统已具备项目、任务、验收、结算的主链路能力，并完成阶段2收口。
- 阶段3启动重点是把“已落地能力”转化为“稳定运营节奏”，补齐云端真链路验证与跨角色协同机制。
- 页面视觉延续现有风格，仅新增必要的状态提示与运营入口，不做大规模界面翻新。

## Core Features

- 阶段3周计划与里程碑分解（按周可执行）。
- 协作矩阵（你+AI）与依赖链可追踪。
- 云端真链路回归清单（项目、任务、验收、结算、审计）与通过标准。
- 关键风险与阻断项（含云端鉴权）闭环处理机制。
- 周度评审证据包模板与发布节奏（可持续复盘）。

## Implementation Approach

采用“启动基线固化 → 真链路验证 → 运营机制上线”的三段法：

1. 固化阶段2基线与阶段3范围，形成可分派清单；
2. 先解决云端鉴权与真链路回归，再扩展阶段3功能节奏；
3. 用周度证据包驱动评审，避免仅靠口头进度。
   关键决策：复用现有 `App.tsx` 编排与 repository 模式，不引入新框架；优先补齐可观测与责任闭环，降低上线不确定性。
   性能与可靠性：延续现有前端筛选管道复杂度 O(n)~O(n log n)；云端写入保持幂等键策略，避免重复写入与审计漂移。
