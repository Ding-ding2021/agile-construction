## 产品概述

根据 PRD 文档补齐项目管理页面的字段信息，使列表展示符合业务需求规范。

## 核心功能需求

1. **列表字段补全**：在现有项目列表中增加品牌/门店、当前阶段、计划开业时间、风险等级字段
2. **统计卡片调整**：将现有的"门店项目/建设中/已完成/风险预警"调整为 PRD 标准的"全部项目/执行中/待验收/风险预警"
3. **数据结构调整**：扩展 ProjectItem 类型定义，增加 PRD 要求的业务字段

## 字段变更详情

| 字段类型     | 新增/调整内容                                                |
| ------------ | ------------------------------------------------------------ |
| 品牌/门店    | 新增品牌名称展示                                             |
| 当前阶段     | 新增项目阶段（立项准备/设计准备/施工实施/验收交付/结算归档） |
| 计划开业时间 | 新增关键业务时间字段                                         |
| 风险等级     | 扩展风险展示，从仅数量增加等级标识（低/中/高/严重）          |
| 统计口径     | 门店项目→全部项目，建设中→执行中，已完成→待验收              |

## 技术约束

- 保持现有样式体系（颜色、间距、圆角）
- 表格列宽需要重新分配以适应新增字段
- 风险等级需要颜色编码（绿色/黄色/橙色/红色）

## 技术栈

- React + TypeScript
- 现有 CSS 样式体系（基于 BEM 命名规范）
- 静态数据 mock

## 实现方案

### 列表字段扩展策略

1. **数据结构扩展**：在 ProjectItem 类型中新增 `brand`、`stage`、`plannedOpenDate`、`riskLevel` 字段
2. **表格列布局调整**：

- 项目名称列宽度从 32% 压缩至 24%
- 进度列保持 24%
- 新增品牌列 8%
- 新增阶段列 10%
- 里程碑列 6%、任务列 8%、风险列 8%、负责人列 8%、操作列 4%

3. **风险等级展示**：将纯数字风险数改为"等级徽章+数量"组合展示，使用颜色编码区分等级

### 统计卡片调整策略

1. 修改标签文案：门店项目→全部项目、建设中→执行中、已完成→待验收
2. 保持风险预警不变
3. 保持现有的四种色调（blue/green/purple/orange）

### 样式调整

1. 为风险等级徽章新增四种状态样式（low/medium/high/critical）
2. 为阶段标签新增样式类
3. 保持响应式布局，确保表格横向滚动体验
